## Development Setup

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/fashion.git`
3. Create a branch: `git checkout -b feature/your-feature`
4. Follow the setup guide in [docs/SETUP.md](docs/SETUP.md)

## Code Style

### Frontend (TypeScript/React)
- Use TypeScript for all new files
- Follow ESLint rules
- Use functional components with hooks
- Prefer Tailwind CSS for styling

### Backend (Python)
- Follow PEP 8 style guide
- Use type hints
- Document functions with docstrings
- Use FastAPI best practices

## Commit Messages

Use clear, descriptive commit messages:
```
feat: Add product search functionality
fix: Resolve image generation timeout issue
docs: Update API documentation
refactor: Improve prompt enhancement logic
```

## Pull Request Process

1. Ensure all tests pass
2. Update documentation if needed
3. Add tests for new features
4. Request review from maintainers
5. Address feedback promptly

## Testing

- Write tests for new features
- Ensure existing tests still pass
- Test on multiple browsers/devices for frontend changes

## Questions?

Open an issue or contact the maintainers.
